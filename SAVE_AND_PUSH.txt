STEP 1: SAVE YOUR FILE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
In your code editor (VS Code, Cursor, etc.):
1. Press Ctrl+S (or Cmd+S on Mac)
2. Make sure the file is saved (no dot/asterisk in the tab)


STEP 2: PUSH TO STAGING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Option A: Use the Script (Easiest)
───────────────────────────────────
1. Double-click: push_to_staging.bat
2. Confirm you want to push
3. Enter a commit message (e.g., "Update calculator logic")
4. Done!


Option B: Manual Commands
──────────────────────────
Open terminal in fpcalculator folder and run:

git checkout staging
git add .
git commit -m "Your commit message here"
git push origin staging


STEP 3: TEST ON STAGING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Wait 1-2 minutes for Streamlit Cloud to redeploy
2. Visit: YOUR_USERNAME-fpcalculator-staging.streamlit.app
3. Test your changes
4. Make sure everything works


STEP 4: PROMOTE TO PRODUCTION (When Ready)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
When staging looks good:
1. Double-click: promote_to_production.bat
2. Your changes go live!


TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

"No changes to commit"
→ Make sure you saved your file (Ctrl+S)
→ Check if the file has a dot/asterisk in the tab

"Already on staging"
→ That's fine! Just continue with git add, commit, push

"Merge conflict"
→ The script will guide you through resolving it

"Permission denied"
→ Make sure you're authenticated with GitHub


QUICK REFERENCE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Check current branch:
git branch --show-current

See what changed:
git status

See specific changes:
git diff false_positive_calculator.py

Undo changes (before commit):
git checkout -- false_positive_calculator.py
